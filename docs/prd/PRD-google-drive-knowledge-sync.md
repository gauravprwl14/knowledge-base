## Business Context

A user connects their Google Drive and expects KMS to become a searchable, queryable knowledge base of their files — including video recordings of meetings, lectures, and presentations. Today, connecting Google Drive syncs *everything* with no control over which folders matter, no transcription of video/audio, no semantic search with source citations, and no way to clean the slate and start over.

This PRD defines the complete end-to-end experience: selective folder sync, background video transcription queue, hybrid search with chunk-level citations powered by a local LLM, and a reset/disconnect flow that removes all indexed data.

## User Stories

### US-01 — Selective Folder Sync

> As a knowledge-worker,
> I want to choose which Google Drive folders to sync,
> so that only relevant work files are indexed (not personal photos, archived projects, etc.)

### US-02 — Video Transcription Queue

> As a knowledge-worker,
> I want KMS to automatically detect video/audio files in my synced Drive folders and transcribe them in the background,
> so that meeting recordings and lecture videos become searchable text without any manual work from me.

### US-03 — Transcription Control (Include / Exclude)

> As a knowledge-worker,
> I want to define rules for which files get transcribed (e.g., only `.mp4` over 1 minute, skip files with "IGNORE" in the name),
> so that I don't waste processing time on irrelevant recordings.

### US-04 — Semantic Search with Citations

> As a knowledge-worker,
> I want to type a natural language question and get relevant document chunks in response, each with a clickable source link and the exact quoted snippet,
> so that I can verify answers and navigate directly to the source.

### US-05 — Local LLM Answer Generation

> As a knowledge-worker,
> I want to ask questions and get synthesised answers from my documents, generated by a local LLM (Ollama),
> so that my data never leaves my infrastructure.

### US-06 — Reset & Clear

> As a knowledge-worker,
> I want to disconnect a source and choose whether to delete all indexed files/vectors or keep them,
> so that I can start fresh without residual stale data polluting my search results.

## Scope

### In Scope (this PRD)

- Folder picker UI for Google Drive sync (multi-select Drive folders)
- Folder filter persistence in `KmsSource.configJson`
- Scan worker respects folder filter (scans only selected folders)
- File type filter: include/exclude MIME types or extensions per source
- Video/audio detection in embed-worker → publishes to `kms.voice` queue
- Voice-app integration with embed pipeline (transcript → chunks → Qdrant)
- Transcription control: per-source include/exclude rules stored in DB
- Background transcription job queue with status UI
- Real-mode BM25 search (PostgreSQL FTS + ts_rank)
- Real-mode semantic search (Qdrant ANN with query embedding)
- Hybrid search with RRF (already implemented as RRF, needs real backends)
- Search result citations: chunk snippet, filename, page/timestamp link
- Ollama LLM answer generation (already implemented in rag-service, needs wiring)
- `POST /sources/:id/disconnect` with `?clearData=true` option
- Async data clear: removes kms_files, kms_chunks (Qdrant), kms_transcription_links for source

### Out of Scope

- Google Workspace docs (Docs/Sheets/Slides) export — deferred: requires separate Google Drive export API, significant complexity
- Real-time Drive webhooks (push notifications) — deferred: requires public HTTPS callback URL, complex lifecycle
- Semantic deduplication — deferred to M6
- Multi-provider transcription fallback (Groq, Deepgram) — deferred: Whisper covers MVP
- GPU acceleration for embedding or transcription — deferred: CPU sufficient for initial scale
- Obsidian or local folder changes — this PRD is Google Drive only
- User-level LLM configuration (model picker) — deferred to settings feature
- Citation highlighting inside document viewer — deferred: requires PDF.js integration
- Scheduled/automatic scans (cron) — deferred to M6

## Feature Breakdown by Component

### Feature 1: Selective Folder Sync

**What exists**: Scan-worker paginates entire Drive (`trashed=false`, no folder filter).
**What's needed**: User picks folders; scan-worker scans only those folders' files.

**Data model addition**:
```
KmsSource.configJson: {
  // Existing:
  "path": "...",   // local only

  // New (Google Drive):
  "syncFolderIds": ["folder-id-1", "folder-id-2"],   // [] = sync entire drive
  "includeExtensions": [],   // [] = include everything
  "excludeExtensions": [],   // e.g. [".dmg", ".zip"]
  "transcribeVideos": true,
  "transcriptionMinDurationSecs": 60,   // skip clips under 1 minute
  "transcriptionExcludePatterns": []    // e.g. ["IGNORE", "test"]
}
```

**Drive folder listing**: `GET /sources/google-drive/folders` endpoint
— Returns folder tree for authenticated Drive account
— Paginated depth-first, max 3 levels

**Scan worker change**: When `syncFolderIds` is non-empty, add `q` filter:
```
q = "trashed=false AND ({parent_filter})"
parent_filter = " OR ".join([f"'{id}' in parents" for id in syncFolderIds])
```

### Feature 2: Video Transcription Pipeline

**What exists**: `voice-app` service is complete (Whisper, AMQP consumer, DB). NOT deployed.
**What's needed**:
1. `voice-app` added to `docker-compose.prod.yml`
2. `embed-worker` detects audio/video MIME types → publishes to `kms.voice`
3. Voice-app publishes transcript result back to `kms.embed`
4. Transcript chunks indexed in Qdrant with `source_type: "voice_transcript"`

**Audio/video MIME types to trigger transcription**:
```
audio/mpeg, audio/wav, audio/ogg, audio/mp4, audio/webm, audio/flac,
video/mp4, video/quicktime, video/webm, video/x-msvideo, video/mpeg
```

**Transcription control rules** (evaluated in embed-worker before publishing):
1. `voiceTranscription.enabled` feature flag must be `true`
2. `transcribeVideos` must be `true` in source configJson
3. File size ≤ `voiceTranscription.maxFileSizeMb` (default 500MB)
4. Duration (from mutagen metadata) ≥ `transcriptionMinDurationSecs` (default 60s)
5. Filename does NOT match any pattern in `transcriptionExcludePatterns`

**Transcription job status** visible in file detail view:
- `PENDING` → Waiting in queue
- `PROCESSING` → Whisper running
- `COMPLETED` → Transcript available
- `FAILED` → Error with message
- `SKIPPED` → Did not meet filter rules

### Feature 3: Real Hybrid Search with Citations

**What exists**: RRF fusion code ✅. BM25 real-mode ❌ (stub). Semantic real-mode ❌ (stub). Citation snippets ❌ (never populated).
**What's needed**:

**BM25 real mode** (search-api):
```sql
SELECT f.id, f.original_filename, c.id as chunk_id, c.content,
       ts_rank(c.search_vector, query) as score,
       ts_headline('english', c.content, query, 'MaxWords=30') as snippet
FROM kms_chunks c
JOIN kms_files f ON f.id = c.file_id
, plainto_tsquery('english', $1) query
WHERE c.user_id = $2
  AND c.search_vector @@ query
  AND ($3::uuid[] IS NULL OR f.source_id = ANY($3))
ORDER BY score DESC
LIMIT $4
```

**Semantic real mode** (search-api):
1. POST query text to embed-worker `/embed` endpoint → get 1024-dim vector
2. POST to Qdrant `/collections/kms_chunks/points/search` with `user_id` payload filter
3. Return top-k points with `content` payload as snippet

**Citation shape** (fully populated):
```typescript
Citation {
  file_id: string,
  filename: string,
  snippet: string,        // exact chunk text, up to 300 chars
  score: number,          // [0.0, 1.0]
  chunk_index: number,    // position in file (for ordering)
  web_view_link?: string, // Drive link (if Google Drive source)
  timestamp_secs?: number // video timestamp (if voice transcript chunk)
}
```

**Timestamp citations for transcripts**: When chunk comes from a voice transcript, store `start_secs` in Qdrant payload. Citation links directly to video timestamp.

### Feature 4: Local LLM Answer Generation (Ollama)

**What exists**: `generator.py` in rag-service implements Ollama streaming via HTTP. `LLMFactory` exists.
**What's needed**: Wire `LLMFactory` into `chat.py` endpoint (currently uses hardcoded `Generator` class).

**Flow**:
```
POST /chat/completions { question, session_id, top_k }
  → TierRouter.route(question)
    → Tier 1: BM25 (fast)
    → Tier 2: Hybrid (if needed)
  → LLMGuard.should_generate(results)
    → If high confidence: return citations only (no LLM)
    → If low confidence: invoke LLM
  → LLMFactory.get_provider()
    → AnthropicProvider (if ANTHROPIC_API_KEY set)
    → OllamaProvider (fallback / local-only)
  → Stream SSE: tokens + sources + done
```

**Ollama requirement**: Ollama must be running at `OLLAMA_BASE_URL` with `OLLAMA_MODEL` loaded.
**Local-only mode**: Set `ANTHROPIC_API_KEY=` (empty) → forces Ollama.

### Feature 5: Reset & Clear

**What exists**: `DELETE /sources/:id` disconnects source but does NOT delete indexed files.
**What's needed**: Explicit clear option.

**API change**:
```
DELETE /sources/:id?clearData=true
```

**Clear data flow** (async background job):
1. Disconnect source (mark status = DISCONNECTED)
2. List all `kms_files` for source
3. Delete Qdrant points for all file_ids in batch
4. Delete `kms_chunks` rows for all file_ids
5. Delete `kms_transcription_links` rows
6. Delete `kms_voice_jobs` rows (via file_id or source_id)
7. Delete `kms_files` rows
8. Delete `kms_scan_jobs` rows
9. Clear Redis delta-sync page token: `DEL kms:scan:drive_page_token:{sourceId}`
10. Mark source as deleted (soft delete or hard delete)

**Progress tracking**: `GET /sources/:id/clear-status` → `{ status: "clearing" | "done", filesCleared, chunksCleared }`

**UI confirmation**: Two-step confirm modal:
- "Disconnect only" — source removed, indexed content stays searchable
- "Disconnect + Clear All Data" — removes all indexed files, chunks, vectors

## Acceptance Criteria

### AC-01: Selective Folder Sync — Happy Path
```gherkin
Given a user has connected their Google Drive
When they open the source settings and select 2 folders to sync
And trigger a scan
Then the scan-worker queries only files with one of the selected folder IDs as parent
And files in other folders are not created in kms_files
And the UI shows "Syncing 2 folders"
```

### AC-02: Selective Folder Sync — Validation
```gherkin
Given a user tries to save folder selection with 0 folders and syncFolderIds = []
Then the system treats [] as "sync entire drive"
And the UI shows "Syncing all folders" as the label
```

### AC-03: Video Transcription — Auto-Trigger
```gherkin
Given voiceTranscription.enabled = true AND transcribeVideos = true in source config
And a scan discovers a video/mp4 file larger than 1 minute
When the embed-worker processes the file
Then a VoiceJobMessage is published to the kms.voice queue
And a kms_voice_jobs record is created with status = PENDING
And the file detail UI shows "Transcription: Queued"
```

### AC-04: Video Transcription — Filter Rules
```gherkin
Given transcriptionMinDurationSecs = 60 AND a video file is 30 seconds long
When the embed-worker evaluates the file
Then it does NOT publish to kms.voice
And the file status shows "Transcription: Skipped (too short)"
```

### AC-05: Video Transcription — Background Processing
```gherkin
Given 10 video files are queued
When the voice-app worker processes them
Then it processes them one at a time (prefetch = 1)
And each transcript is published back to kms.embed
And each transcript's chunks are upserted to Qdrant with source_type = "voice_transcript"
And the file's transcription status updates to COMPLETED
```

### AC-06: Semantic Search — Real Mode
```gherkin
Given MOCK_SEMANTIC = false AND embedding.enabled = true
When a user searches "meeting agenda Q2 planning"
Then the search-api generates a query embedding via embed-worker
And queries Qdrant with user_id filter
And returns up to 10 results with snippet, score, and web_view_link
Within 2 seconds (p95)
```

### AC-07: Citations — Chunk Snippets
```gherkin
Given a search returns 3 results
Then each result includes:
  - filename (non-empty)
  - snippet (up to 300 chars of actual chunk text)
  - score (0.0–1.0)
  - web_view_link (Drive link, if available)
  - timestamp_secs (for voice transcript chunks)
```

### AC-08: Local LLM — Ollama Answer
```gherkin
Given ANTHROPIC_API_KEY is empty AND OLLAMA_BASE_URL is reachable with llama3.2:3b loaded
When a user asks "summarise my Q1 strategy documents"
Then the rag-service retrieves top chunks via hybrid search
And calls Ollama with a context-stuffed prompt
And streams the answer back via SSE
And includes citations at the end of the stream
```

### AC-09: Reset — Disconnect Only
```gherkin
Given a source is CONNECTED
When a user clicks "Disconnect" without checking "Clear all data"
Then the source status is set to DISCONNECTED
And kms_files for the source remain in the DB
And search results still include those files
```

### AC-10: Reset — Full Clear
```gherkin
Given a source is CONNECTED with 500 indexed files
When a user clicks "Disconnect + Clear All Data" and confirms
Then the source is disconnected
And all 500 kms_files are deleted
And all Qdrant points for those files are deleted
And the source's Redis delta-sync token is cleared
And GET /sources shows 0 files for that source
And search returns no results from that source
```

### AC-11: Transcription Search
```gherkin
Given a video file "Q1_kickoff.mp4" has been transcribed
And the transcript contains "OKR targets for next quarter"
When a user searches "OKR targets"
Then the search results include a citation from "Q1_kickoff.mp4"
With the snippet showing the relevant transcript text
And a timestamp_secs that links to the correct moment in the video
```

## API Contracts

### New/Changed Endpoints

```
# Folder tree (OAuth-authenticated drive)
GET /sources/google-drive/folders?sourceId={id}&parentId=root
→ { folders: [{ id, name, path, fileCount, children: [...] }] }

# Update source config (folder selection + filter rules)
PATCH /sources/:id/config
Body: {
  syncFolderIds?: string[],
  includeExtensions?: string[],
  excludeExtensions?: string[],
  transcribeVideos?: boolean,
  transcriptionMinDurationSecs?: number,
  transcriptionExcludePatterns?: string[]
}
→ { source: KmsSource }

# Disconnect with optional data clear
DELETE /sources/:id?clearData=false
→ 204 No Content (disconnect only, fast)
DELETE /sources/:id?clearData=true
→ 202 Accepted { jobId: string } (async clear, may take seconds)

# Clear progress
GET /sources/:id/clear-status
→ { status: "clearing"|"done"|"failed", filesCleared, chunksCleared, vectorsCleared }

# Transcription status per file
GET /files/:id/transcription
→ { status, jobId, transcript?, language?, durationSecs?, completedAt? }
```

### Modified Search Response

```typescript
interface SearchResult {
  id: string           // chunk ID
  fileId: string
  filename: string
  content: string      // chunk text (snippet) — PREVIOUSLY MISSING
  score: number
  chunkIndex: number
  webViewLink?: string // Drive link — NEW
  timestampSecs?: number // for voice transcripts — NEW
  sourceType?: string  // "google_drive" | "voice_transcript" | "local"
}
```
